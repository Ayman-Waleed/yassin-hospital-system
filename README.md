# نظام العيادات الخارجية — مستشفى الياسين الطبي

مشروع تخرج: موقع ويب متكامل للعيادات الخارجية يتكون من **موقع للزوار/المرضى** (حجز مواعيد، استعراض الأطباء والعيادات، شكاوى، إعلانات) و**لوحة تحكم للإدارة** (إدارة كاملة للأطباء والعيادات والمواعيد والشكاوى والمستخدمين).

## التقنيات المستخدمة

| الطبقة | التقنية |
|---|---|
| الواجهة الأمامية | HTML / CSS / JavaScript (بدون frameworks) |
| الخادم | Node.js + Express |
| قاعدة البيانات | MySQL (بترميز utf8mb4 لدعم العربية) |
| مكتبات رئيسية | mysql2 (اتصال Pool + Prepared Statements)، multer (رفع الصور)، dotenv، cors |

## هيكلية المشروع

```
Al-Yaseen/
├── server.js                 نقطة دخول السيرفر
├── .env                      إعدادات الاتصال (انسخه من .env.example)
├── database/
│   ├── schema.sql            إنشاء القاعدة والجداول (7 جداول)
│   └── seed.sql              بيانات تجريبية واقعية (12 عيادة، 12 طبيباً، دوام، مواعيد...)
├── src/
│   ├── config/database.js    اتصال MySQL (Pool + utf8mb4)
│   ├── models/               استعلامات قاعدة البيانات لكل مورد
│   ├── controllers/          منطق كل مورد + التحقق من المدخلات
│   └── routes/               مسارات الـ API لكل مورد
├── public/
│   ├── site/                 موقع الزوار (الرئيسية، الحجز، الأطباء، العيادات...)
│   └── admin/                لوحة تحكم الإدارة
└── uploads/                  صور الأطباء المرفوعة
```

## خطوات التشغيل من الصفر

### 1) المتطلبات
- تنصيب [Node.js](https://nodejs.org) (إصدار 18 أو أحدث)
- تنصيب [XAMPP](https://www.apachefriends.org) وتشغيل خدمة **MySQL** من لوحة XAMPP

### 2) تنصيب الاعتماديات
افتح الطرفية داخل مجلد المشروع ونفّذ:
```bash
npm install
```

### 3) إنشاء قاعدة البيانات واستيرادها
> ⚠️ **ملاحظة مهمة:** استيراد `seed.sql` عبر phpMyAdmin قد يفشل بسبب قيود المفاتيح الأجنبية.
> الطريقة المضمونة هي سطر الأوامر (الملف يحتوي أصلاً على `SET FOREIGN_KEY_CHECKS=0` قبل التفريغ):

```bash
# من مجلد المشروع (عدّل مسار mysql حسب جهازك إن لم يكن في PATH)
C:\xampp\mysql\bin\mysql -u root < database\schema.sql
C:\xampp\mysql\bin\mysql -u root nasser_hospital < database\seed.sql
```

### 4) إعداد ملف البيئة
انسخ `.env.example` باسم `.env` وعدّل القيم إذا كانت إعدادات MySQL عندك مختلفة:
```
DB_HOST=localhost
DB_USER=root
DB_PASSWORD=
DB_NAME=nasser_hospital
PORT=5000
SESSION_SECRET=اكتب-سلسلة-عشوائية-طويلة
DB_SSL=false
```

### 5) تشغيل المشروع
```bash
npm start          # تشغيل عادي
npm run dev        # تشغيل بالتحديث التلقائي (nodemon) — يُنصح به أثناء التطوير
```

### 6) فتح الموقع
| القسم | الرابط |
|---|---|
| موقع الزوار | http://localhost:5000/site/index.html |
| لوحة الإدارة | http://localhost:5000/admin/index.html |
| فحص اتصال القاعدة | http://localhost:5000/check-db |

> 💡 افتح الموقع دائماً من **localhost:5000** (سيرفر Node) وليس من Live Server —
> ومع ذلك، لو فُتحت أي صفحة من بورت آخر (مثل 5500) ستُحوَّل تلقائياً للبورت الصحيح.

## بيانات الدخول التجريبية

**لوحة الإدارة:**
- اسم المستخدم: `admin`
- كلمة المرور: `123`

## أبرز الميزات التقنية

- **حجز ذكي مرتبط بدوام الأطباء**: الأوقات المتاحة تُحسب من جدول `doctor_schedules` مع استبعاد المحجوز، والتحقق النهائي يتم على السيرفر (وقت خارج الدوام يُرفض، والحجز المزدوج يمنعه فهرس فريد في القاعدة).
- **تذكرة حجز برقم حقيقي + QR**: رقم التذكرة هو معرف الحجز في القاعدة.
- **Prepared Statements في كل الاستعلامات** — حماية من SQL Injection.
- **رسائل خطأ عربية** بصيغة موحدة `{ success, message, data }` في المسارات الجديدة.
- دورة شكوى كاملة: الزائر يرسل ← الإدارة ترد ← الزائر يتابع الرد بهاتفه من صفحة الشكاوى.

## إدارة الصور

نظام الصور يعمل بأولوية ثلاثية لأي طبيب أو عيادة:
1. **الصورة المرفوعة من لوحة الإدارة** (`image_url` في القاعدة) — إن وُجدت تفوز دائماً.
2. وإلا **الصورة البديلة** باسم العنصر في `public/site/pictures/` (بدائل SVG احترافية جاهزة).
3. وإلا **الصورة الافتراضية** (`default-doctor.svg` / `default-clinic.svg` / `default.svg`) — فأي عنصر جديد بلا صورة يظهر مرتباً تلقائياً، لا توجد صور مكسورة أبداً.

### رفع صورة طبيب أو عيادة من لوحة الإدارة
1. افتح **إدارة الأطباء** أو **إدارة العيادات** ← إضافة أو تعديل.
2. في المودال ستجد حقل **"صورة الطبيب/العيادة (اختياري)"** — اختر ملف صورة واحفظ.
3. تُحفظ الصورة في مجلد `uploads/` باسم فريد، ويُخزن مسارها في القاعدة، وتظهر فوراً في موقع الزوار.
- ملاحظة: عند التعديل بدون اختيار صورة جديدة تبقى الصورة القديمة كما هي.

### استبدال صورة بديلة بصورة حقيقية يدوياً
ضع ملف الصورة في `public/site/pictures/` **بنفس الاسم** و**بأي صيغة شائعة** — `jpg` أو `jpeg` أو `png` أو `webp` — وستظهر فوراً دون تعديل أي كود:
السيرفر يجرب الصيغ بالترتيب تلقائياً (jpg ← jpeg ← png ← webp ← البديل svg)، فلا يهم بأي امتداد تطلبها الصفحة.

**كيف يعمل ذلك؟** middleware في `server.js` يقدّم الملف الحقيقي إن وُجد بأي صيغة؛ وإن كان مفقوداً يقدّم النسخة البديلة `SVG` بنفس الاسم؛ وإن لم توجد بديلة تتكفل آلية `onerror` في الواجهة بعرض الافتراضية. الأسماء العربية والمسافات مدعومة بالكامل (مختبرة على Windows وفي المتصفح).

### جدول الصور المرجعي الكامل
كل الملفات توضع في `public/site/pictures/` بالاسم المذكور حرفياً (الامتداد: أي صيغة شائعة):

| العنصر | اسم الملف | المقاس المقترح (بكسل) |
|---|---|---|
| صورة كل طبيب في القاعدة | اسم الطبيب بدون "د. " — مثال `أحمد مرتجى.jpg` | 360×420 عمودية (6:7) |
| أطباء سلايدر الرئيسية | `احمد جميل` `احمد محمد` `احمد محمود` `احمد مرتجى` `ايهاب احمد` `عرفات مرزوق` `محمد صقر` `مرتضى يونس` `مرزوق الفرا` `نور مرتجى` `ياسين محمد` `يونس اشرف` | 360×420 عمودية (6:7) |
| صورة كل عيادة | اسم العيادة حرفياً — مثال `عيادة القلب.jpg` | 900×500 عرضية (~16:9)، الموضوع في المنتصف* |
| صورة كل إعلان | عنوان الإعلان حرفياً — مثال `تحسين خدمات الاستقبال.jpg` | 800×450 عرضية (16:9) |
| إعلانات الرئيسية الثابتة | `اعلان 1` · `اعلان2` · `اعلان 3` | 800×450 عرضية (16:9) |
| اللوجو (هيدر + فوتر) | `لوجو الموقع.png` | ~500×160 عرضية (3:1) بخلفية شفافة |
| خلفية الأبطال (hero) | `الرئيسية 1.png` | 1920×700 عريضة جداً |
| صورة المستشفى | `المستشفى.jpg` | 800×500 عرضية |
| صورة "من نحن" الدائرية | `تجمع من نحن.jpg` | 500×500 مربعة (تُعرض دائرة) |
| إشعار الحجز | `اشعار الحجز.png` | 400×400 مربعة |
| بطاقة العيادات العامة | `عيادات.jpg` | 800×500 عرضية |
| شخصية الطبيب / الطبيبة | `الدكتور.png` · `دكتورة.png` | 600×800 عمودية بخلفية شفافة |
| الأسئلة الشائعة | `الأسئلة الشائعة.jpg` | 800×500 عرضية |

## الأمان (جاهز للنشر العام)

- **مصادقة حقيقية على السيرفر**: دخول الإدارة عبر `POST /api/auth/login` يتحقق من جدول `users` بكلمات مرور مشفرة **bcrypt**، وينشئ جلسة **express-session** بكوكي HttpOnly.
- **كل المسارات الإدارية محمية بـ middleware** (`requireAdmin`): الإضافة والتعديل والحذف والرد على الشكاوى وقوائم البيانات الشخصية (كل المواعيد/كل الشكاوى/المستخدمون) ترفض أي طلب بلا جلسة بـ401 — حتى لو استدعاها أحد مباشرة من خارج الواجهة.
- **المتاح للزوار فقط**: تصفح العيادات والأطباء والدوام والإعلانات، حجز موعد، إرسال شكوى، ومتابعة رسائله بهاتفه.
- **حدود معدل الطلبات**: 10 محاولات دخول/ربع ساعة، و30 حجزاً أو شكوى/ساعة لكل جهاز (ضد التخمين والسبام).
- **تعقيم HTML** لكل النصوص القادمة من الزوار قبل عرضها (حماية من XSS المخزن).
- **Prepared Statements** في كل الاستعلامات (حماية من SQL Injection).

## النشر على استضافة مجانية (Render + قاعدة سحابية)

1. **ارفع المشروع على GitHub** (ملف `.gitignore` يستثني `.env` و`node_modules` أصلاً).
2. **قاعدة البيانات**: أنشئ قاعدة MySQL مجانية على [TiDB Cloud Serverless](https://tidbcloud.com) أو [Aiven](https://aiven.io)، ثم استورد `schema.sql` و`seed.sql` عبر عميل mysql بنفس أوامر التشغيل المحلي مع بيانات الاتصال السحابية.
3. **السيرفر**: على [Render](https://render.com) أنشئ **Web Service** من مستودعك — Build: `npm install`، Start: `npm start`.
4. **متغيرات البيئة على Render**: `DB_HOST` و`DB_USER` و`DB_PASSWORD` و`DB_NAME` (من لوحة القاعدة السحابية) + `DB_SSL=true` + `SESSION_SECRET` (سلسلة عشوائية طويلة) + `NODE_ENV=production`.
5. افتح الرابط الذي يعطيك إياه Render — الموقع كامل (زوار + إدارة) على نفس الدومين.

**ملاحظات النشر المجاني:**
- خطة Render المجانية تُنيم السيرفر بعد ~15 دقيقة خمول؛ أول زيارة بعد النوم تأخذ ~30-50 ثانية.
- القرص مؤقت: **الصور المرفوعة من لوحة الإدارة (`uploads/`) تُمسح مع كل إعادة نشر أو إعادة تشغيل** — لن تنكسر أي صفحة (نظام البدائل يغطيها تلقائياً)، لكن للصور الدائمة ضعها يدوياً في `public/site/pictures/` داخل المستودع (انظر جدول الصور)، وسبق أن اخترت هذا الأسلوب.
- النوم يمسح جلسات الدخول — الأدمن يسجل دخوله من جديد ببساطة.

## تحسينات مستقبلية (للمناقشة)

- تخزين الصور المرفوعة على خدمة سحابية (Cloudinary) بدل القرص المؤقت.
- فصل صلاحيات الأدوار (مدير نظام / موظف استقبال) داخل لوحة الإدارة.
- إشعارات للمريض (بريد/رسالة) عند تأكيد الموعد أو الرد على الشكوى.
- ترقيم صفحات (pagination) حقيقي للإعلانات والمواعيد عند كبر البيانات.
